Help

General use

This web based application is written in Django and is intended for users of cars and other fuel based modes of transportation

Each time you tank some fuel, you enter into the application the following details:

  • Date
  • Amount of kilometers driven up to this point
  • Amount of liters of fuel tanked (full volume in metric liters)
  • Cost of fuel tanked (full cost in DKK)

The application will then provide you with an overview of your mileage, total expenses, fuel cost, etc. You can check out these measures in the overview panel

Upload data

If you have some existing data (i.e. in an Excel spreadsheet) you can upload your data in the upload panel. It is important that the format of the data file adheres to the following specification:

  • It must consist of 4 columns
  • The names of the columns must be the strings: 'date', 'km', 'liter', 'cost' (the order is irrelevant), and these must be in the first row
  • The day, month and year must be separated with a forward slash ('/')
  • It is a comma separated value file (i.e. each row is one data entry and each column is separated by a comma).

If the above format is not followed, the application will not be able to process your data in any meaningful way (I'm not a wizard).

Before pressing the 'upload' button, you also need to specify the format of your date column. If your data is of the format day/month/year, you select the dd/mm/yyyy option, and so on.

Once you have successfully uploaded your data, the full data set should be available in the history panel.

A suitable input file can look like the following:

    date, km, amount, liter
    1/1/1970, 0, 100, 10
    1/15/1970, 50, 70, 7
    2/1/1970, 120, 90, 8
